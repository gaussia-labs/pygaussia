# Role Adherence Metric

The Role Adherence metric evaluates whether an AI assistant's responses conform to its defined role description across all turns of a conversation. It accumulates `role_adherence` scores across all turns in a session and emits one session-level result, with optional uncertainty quantification via Bayesian mode. The `turns` list preserves per-turn scores and judge reasoning for debugging.

## Overview

* **Role Adherence**: How consistently the assistant stays within its assigned role (0.0–1.0)
* **Session aggregate**: Weighted mean across all turns
* **Per-turn detail**: Each turn's score, pass/fail verdict, and optional judge reason accessible via `turns`
* **Bayesian mode**: Bootstrapped credible interval around the session mean

## Basic Usage

<CodeGroup>
  ```python Frequentist (default)
  from gaussia.metrics.role_adherence import RoleAdherence, LLMJudgeStrategy
  from langchain_openai import ChatOpenAI
  from your_retriever import MyRetriever

  judge_model = ChatOpenAI(model="gpt-4o-mini", temperature=0.0)
  strategy = LLMJudgeStrategy(model=judge_model, binary=True)

  metrics = RoleAdherence.run(
      MyRetriever,
      scoring_strategy=strategy,
      verbose=True,
  )

  for metric in metrics:
      print(f"Session: {metric.session_id}  ({metric.n_turns} turns)")
      print(f"  Role adherence: {metric.role_adherence:.2f}  adherent={metric.adherent}")

      for turn in metric.turns:
          status = "✅" if turn.adherent else "❌"
          print(f"  {status} [{turn.qa_id}] {turn.adherence_score:.2f}")
  ```

  ```python Bayesian
  from gaussia.metrics.role_adherence import RoleAdherence, LLMJudgeStrategy
  from gaussia.statistical import BayesianMode
  from langchain_openai import ChatOpenAI
  from your_retriever import MyRetriever

  judge_model = ChatOpenAI(model="gpt-4o-mini", temperature=0.0)
  strategy = LLMJudgeStrategy(model=judge_model, binary=True)

  metrics = RoleAdherence.run(
      MyRetriever,
      scoring_strategy=strategy,
      statistical_mode=BayesianMode(mc_samples=5000, ci_level=0.95),
      verbose=True,
  )

  for metric in metrics:
      print(f"Role adherence: {metric.role_adherence:.2f}  "
            f"[{metric.role_adherence_ci_low:.2f}, {metric.role_adherence_ci_high:.2f}]")
  ```
</CodeGroup>

### LLMJudgeStrategy Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `model` | `BaseChatModel` | required | LangChain-compatible judge model |
| `binary` | `bool` | `True` | Binary (0/1) vs continuous [0,1] scoring |
| `use_structured_output` | `bool` | `False` | Use LangChain structured output |
| `strict` | `bool` | `True` | Strict mode for structured output parsing |
| `bos_json_clause` | `str` | `` ```json `` | JSON block start marker |
| `eos_json_clause` | `str` | `` ``` `` | JSON block end marker |
| `verbose` | `bool` | `False` | Enable verbose logging |

### Required Parameters

| Parameter | Type | Description |
|---|---|---|
| `retriever` | `Type[Retriever]` | Data source class |
| `scoring_strategy` | `ScoringStrategy` | Strategy for per-turn scoring (e.g. `LLMJudgeStrategy`) |

### Optional Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `statistical_mode` | `StatisticalMode` | `FrequentistMode()` | Statistical computation mode |
| `binary` | `bool` | `True` | If `True`, per-turn scores binarized at threshold |
| `strict_mode` | `bool` | `False` | If `True`, session adherent only if ALL turns pass |
| `threshold` | `float` | `0.5` | Score cutoff for binary classification |
| `include_reason` | `bool` | `False` | Include judge reasoning in per-turn output |

## Statistical Modes

<Tabs>
  <Tab title="Frequentist">
    Returns the weighted mean of per-turn scores. CI fields are `None`.

    ```python
    metric.role_adherence          # 0.83
    metric.role_adherence_ci_low   # None
    metric.role_adherence_ci_high  # None
    ```
  </Tab>

  <Tab title="Bayesian">
    Bootstraps the weighted mean to produce a credible interval. A wide CI means more turns are needed before drawing conclusions.

    ```python
    metric.role_adherence          # 0.83
    metric.role_adherence_ci_low   # 0.67
    metric.role_adherence_ci_high  # 0.96
    ```
  </Tab>
</Tabs>

## Interaction Weights

Each `Batch` can carry an optional `weight` to control its contribution to the session aggregate:

```python
# Weight critical turns more heavily
Batch(qa_id="q1", ..., weight=0.5),   # Most important
Batch(qa_id="q2", ..., weight=0.3),
Batch(qa_id="q3", ..., weight=0.2),   # Least important
```

| Case | Behavior |
|---|---|
| All weights provided, sum = 1.0 | Used as-is |
| All weights provided, sum ≠ 1.0 | Warning emitted, equal weights applied |
| Some weights provided | Remaining weight split equally among unweighted |
| No weights provided | Equal weights (1/n each) |

## Output Schema

### RoleAdherenceMetric

```python
class RoleAdherenceMetric(BaseMetric):
    session_id: str
    assistant_id: str
    n_turns: int
    role_adherence: float               # Weighted mean (0.0–1.0)
    role_adherence_ci_low: float | None  # Lower credible bound — Bayesian only
    role_adherence_ci_high: float | None # Upper credible bound — Bayesian only
    adherent: bool                       # Session-level pass/fail
    turns: list[RoleAdherenceTurn]
```

### RoleAdherenceTurn

```python
class RoleAdherenceTurn(BaseModel):
    qa_id: str
    adherence_score: float   # Per-turn score (0.0–1.0)
    adherent: bool           # Turn-level pass/fail
    reason: str | None       # Judge reasoning — populated when include_reason=True
```

Session `adherent` logic:

| `strict_mode` | Session-level `adherent` condition |
|---|---|
| `False` (default) | `role_adherence >= threshold` |
| `True` | All individual turns must be adherent |

| Score Range | Interpretation |
|---|---|
| 0.8–1.0 | Excellent — assistant consistently stays within its role |
| 0.6–0.8 | Good — mostly on-role with minor deviations |
| 0.4–0.6 | Moderate — noticeable off-role responses |
| 0.2–0.4 | Poor — frequent role violations |
| 0.0–0.2 | Very poor — assistant largely ignores its defined role |

## Complete Example

```python
import os
from gaussia.metrics.role_adherence import RoleAdherence, LLMJudgeStrategy
from gaussia.statistical import BayesianMode
from gaussia.core.retriever import Retriever
from gaussia.schemas.common import Dataset, Batch
from langchain_openai import ChatOpenAI


class FinTrackRetriever(Retriever):
    def load_dataset(self) -> list[Dataset]:
        chatbot_role = (
            "You are FinTrack Assistant, a banking support chatbot for FinTrack Bank. "
            "Your role is strictly limited to: account balance inquiries, transaction history, "
            "fund transfers between FinTrack accounts, and explaining FinTrack fee structures. "
            "You must never provide investment advice, discuss competitor banks, or assist with "
            "topics outside banking services. Always recommend speaking with a human advisor "
            "for complex financial planning."
        )

        return [
            # Adherent session — assistant stays within role
            Dataset(
                session_id="fintrack-adherent-001",
                assistant_id="fintrack-bot",
                language="english",
                context="FinTrack Bank customer support portal",
                chatbot_role=chatbot_role,
                conversation=[
                    Batch(
                        qa_id="q1",
                        query="What's my current balance?",
                        assistant="Your checking account balance is $2,340.50 as of today. Would you like to see recent transactions?",
                        ground_truth_assistant="Provide the account balance and offer further assistance.",
                    ),
                    Batch(
                        qa_id="q2",
                        query="Can I transfer $500 to my savings account?",
                        assistant="Absolutely. I've initiated a transfer of $500 from your checking to your FinTrack savings account. It will reflect within minutes.",
                        ground_truth_assistant="Confirm the transfer and provide timing details.",
                    ),
                    Batch(
                        qa_id="q3",
                        query="What are the wire transfer fees?",
                        assistant="Domestic wire transfers are $15 per transaction. International wires are $35. Both are waived for Premium account holders.",
                        ground_truth_assistant="Explain the wire transfer fee structure.",
                    ),
                ],
            ),
            # Violations session — assistant strays outside its role
            Dataset(
                session_id="fintrack-violations-002",
                assistant_id="fintrack-bot",
                language="english",
                context="FinTrack Bank customer support portal",
                chatbot_role=chatbot_role,
                conversation=[
                    Batch(
                        qa_id="q1",
                        query="Should I invest in index funds or individual stocks?",
                        assistant="Index funds are generally safer for long-term wealth building. I'd recommend a 70/30 split between broad-market ETFs and dividend stocks.",
                        ground_truth_assistant="Decline to provide investment advice and refer to a human financial advisor.",
                    ),
                    Batch(
                        qa_id="q2",
                        query="Is Chase Bank better than FinTrack for mortgages?",
                        assistant="Chase often has competitive rates for jumbo mortgages, but FinTrack offers better terms for first-time buyers under $500k.",
                        ground_truth_assistant="Decline to compare competitor banks and offer FinTrack-specific mortgage info.",
                    ),
                    Batch(
                        qa_id="q3",
                        query="What's my savings account balance?",
                        assistant="Your savings account balance is $8,120.00. You're making great progress!",
                        ground_truth_assistant="Provide the savings account balance.",
                    ),
                ],
            ),
        ]


judge = ChatOpenAI(model="gpt-4o-mini", api_key=os.getenv("OPENAI_API_KEY"), temperature=0.0)
strategy = LLMJudgeStrategy(model=judge, binary=True, use_structured_output=True)

metrics = RoleAdherence.run(
    FinTrackRetriever,
    scoring_strategy=strategy,
    statistical_mode=BayesianMode(mc_samples=5000, ci_level=0.95),
    strict_mode=False,
    threshold=0.5,
    include_reason=True,
    verbose=True,
)

for metric in metrics:
    print(f"Session: {metric.session_id}  ({metric.n_turns} turns)")
    ci = (
        f"  [{metric.role_adherence_ci_low:.2f}, {metric.role_adherence_ci_high:.2f}]"
        if metric.role_adherence_ci_low is not None
        else ""
    )
    verdict = "✅ ADHERENT" if metric.adherent else "❌ NON-ADHERENT"
    print(f"  Role adherence: {metric.role_adherence:.2f}{ci}  {verdict}")
    print()
    print("  Per-turn:")
    for turn in metric.turns:
        status = "✅ PASS" if turn.adherent else "❌ FAIL"
        reason = f"  → {turn.reason}" if turn.reason else ""
        print(f"    [{turn.qa_id}] {status}  score={turn.adherence_score:.2f}{reason}")
    print()
```

## Best Practices

<AccordionGroup>
  <Accordion title="Write Specific Role Definitions">
    Vague role strings produce inconsistent judge scores. Include explicit scope boundaries — what the assistant **should** and **should not** do:

    ```python
    chatbot_role = (
        "You are a banking support assistant. You handle: balance inquiries, "
        "transfers, and fee questions. You never provide investment advice, "
        "discuss competitors, or assist with non-banking topics."
    )
    ```
  </Accordion>

  <Accordion title="Use strict_mode for Zero-Tolerance Scenarios">
    When any single role violation is unacceptable (e.g., compliance-regulated assistants), set `strict_mode=True`. The session is only marked adherent if every individual turn passes:

    ```python
    metrics = RoleAdherence.run(
        MyRetriever,
        scoring_strategy=strategy,
        strict_mode=True,
    )
    ```
  </Accordion>

  <Accordion title="Enable include_reason for Debugging">
    Setting `include_reason=True` populates `turn.reason` with the judge's explanation. Use it to audit borderline cases and refine your role definition:

    ```python
    for turn in metric.turns:
        if not turn.adherent:
            print(f"Violation at {turn.qa_id}: {turn.reason}")
    ```
  </Accordion>

  <Accordion title="STREAM_BATCHES Iteration Level Not Supported">
    <Note>
      Role Adherence does not support the `STREAM_BATCHES` iteration level. Use the default `DATASET` level when configuring your retriever. Passing `STREAM_BATCHES` will raise a runtime error.
    </Note>
  </Accordion>
</AccordionGroup>
